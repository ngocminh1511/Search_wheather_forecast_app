3Khi nào chạy?

NOAA cập nhật dữ liệu 4 lần/ngày (mỗi 6 giờ). Hệ thống fire ngay sau khi NOAA upload xong (~5h delay sau cycle start).

Mỗi mốc giờ, 5 maps fire stagger 3 phút (05:05, 05:08, 05:11, 05:14, 05:17) để không quá tải cùng lúc.

41 bản đồ trải qua 6 bước

Khi đến giờ fire, mỗi bản đồ tuần tự qua các bước sau. Toàn bộ chu kỳ thường mất 20-40 phút/map tùy server.

1

🔍 Hỏi NOAA: "Có dữ liệu mới chưa?"

Gửi HEAD request đến NOAA để xem cycle mới đã upload xong chưa. Nếu chưa, fallback dùng cycle cũ trên disk.

Như check email xem có thư mới không
~5-30 giây
2

⬇️ Tải về 37 file GRIB2 từ NOAA

GRIB2 là format dữ liệu thời tiết raw chứa số liệu cho từng giờ forecast (f000=now, f001=1h sau, ... f036=36h sau).

Như download 37 file Excel chứa số liệu
~2-5 phút
3

🎨 Biến GRIB2 thành tiles PNG

Đọc số liệu → tô màu theo bảng colormap → cắt thành tiles 256×256 px ở nhiều zoom level (z=0..8). Mỗi file GRIB sinh ra hàng nghìn tiles.

Như Photoshop: file raw → ảnh PNG, chia thành lưới ô vuông
~10-25 phút
4

☁️ Đẩy tiles lên Bunny CDN

Upload song song (16-64 connections cùng lúc) cho nhanh. Bunny lưu tiles để các CDN node toàn cầu phục vụ user.

Như upload ảnh lên Google Drive nhưng global
~3-8 phút
5

🔄 Bật công tắc "live"

Ghi file _current.json trên Bunny trỏ vào cycle mới. Cùng lúc user toàn thế giới bắt đầu thấy dữ liệu mới. Atomic = không có khoảnh khắc nửa cũ nửa mới.

Như flip công tắc đèn — bật một phát, sáng cả phòng
~5-30 giây
6

🧹 Dọn dẹp file tạm trên server

Xoá GRIB2 đã tải, tiles staging, file JSON tạm. Đẩy cycle cũ ra khỏi Bunny (giữ 4 cycles gần nhất = 24h). Gửi báo cáo Telegram.

Như xoá file Downloads/temp sau khi xong việc
~1-2 phút

11:05 🌡️ Temperature START → 11:08 🌧️ Rain basic START → 11:11 ⛈️ Rain adv. queued → 11:14 ❄️ Snow queued → 11:17 💨 Wind queued

discovering → downloading 37 GRIB2 files → parse → build → cut tiles → push lên Bunny → flip pointer (user thấy data mới) → cleanup local
Mỗi map ~20-40 phút. Worker pool xử lý parallel: parse/build/cut chạy đồng thời trên các frames khác nhau.
✅ Cycle complete ~11:55 UTC
🌡️ done 🌧️ done ⛈️ done ❄️ done 💨 done → 💬 Per-cycle Telegram report
Cả 5 maps live trên Bunny. Server idle chờ cycle tiếp theo (17:00 UTC).
😴 Idle period ~12:00 → 17:00 UTC
12:30 catchup (skip, đã up-to-date) 14:45 retention (xoá cycle > 24h) 14:30 catchup 16:30 catchup → 17:00 NOAA cycle 12z ready → 17:05 cycle tiếp theo START
Giữa các cycle, server không idle hoàn toàn: catchup/retention nhẹ chạy nền (<1% CPU). Cho các project khác trên cùng server thoải mái dùng CPU/RAM.

6Lỡ có sự cố thì sao?

3 lớp lưới an toàn để pipeline không chết, không mất dữ liệu, không làm server die.

❓ Lỡ NOAA chậm / down?
Catchup job tự kiểm tra mỗi 2 giờ. Nếu thấy có map nào tụt hậu so với cycle mới nhất, nó tự fire lại map đó. Pipeline không bao giờ bị "lỡ chuyến".
Mỗi 2h lúc xx:30
❓ Lỡ disk đầy?
Retention job tự xoá run cũ hơn 4 cycles (24h). Nếu có run kẹt không xoá được lúc finalize, retention vớt sau 6h. Bunny pointer guard chống xoá nhầm data đang live.
Mỗi 6h lúc xx:45
❓ Lỡ server quá tải?
Resource Guard tự throttle worker khi RAM > 85% / CPU > 95% / Disk < 5GB. Project khác trên server không bị giành tài nguyên. Auto-scaler tự giảm worker count nếu peak RAM dự kiến vượt memory limit.
Real-time

7Làm sao biết pipeline đang chạy đúng?

4 cách kiểm tra — chọn cái nào tiện nhất cho bạn.

🟢
Sidebar pill AUTO • ON
Mở admin UI → góc trái trên thấy pill xanh pulse = đang chạy. Đỏ = đã pause. Vàng = lỗi cần restart.
🩺
Tab Diagnostics
7 subsystem cards (DB, scheduler, orchestrator, resources, last cycle, Bunny, NOAA). Tất cả xanh = OK.
💬
Telegram alerts
Tin nhắn sau mỗi map xong + tin tổng hợp sau mỗi cycle + daily report 00:30 UTC. Có lỗi → tin critical ngay lập tức.
📦
Download log bundle
Tab Diagnostics → chọn ngày → Download ZIP. Gửi tôi review nếu nghi sự cố. Bundle < 50MB, chứa logs + DB metrics + state snapshot.

⚙️ Job state machine (PENDING → READY)

Mỗi (map_type × forecast_hour × product) là 1 job. State được lưu vào SQLite table pipeline_jobs:

PENDING
Vừa enqueue, chưa xử lý
PARSING / PARSED
Đang/đã đọc GRIB
BUILDING / BUILT
Đang/đã warp canvas
CUTTING / READY
Đang/đã cắt tiles + push Bunny
SKIPPED
GRIB rỗng/missing — bình thường, không phải lỗi
ERROR
Pool crash hoặc bug — catchup 2h sẽ vớt lại
CANCELLED
User bấm Cancel trong UI
🛡️ Resource thresholds (config có thể tune)
MAX_RAM_PERCENT
85% — throttle khi vượt
MAX_CPU_PERCENT
95% — throttle build/cut khi vượt
MAX_IOWAIT_PERCENT
30% — throttle khi disk IO bottleneck
MIN_DISK_FREE_GB
5GB — pause pipeline khi disk gần đầy
KEEP_CYCLES
4 — giữ 24h data trên Bunny + local
LOG_RETENTION_DAYS
7 — giữ log file 1 tuần
NOAA_RPM_LIMIT
100 req/phút khi probe + download
CPU_BUDGET_PERCENT
50% — share host với project khác (auto-scaler dùng)
